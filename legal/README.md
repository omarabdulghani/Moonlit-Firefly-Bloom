# Legal and IP Documentation

This folder keeps practical internal records for Moonlit Firefly Bloom assets, AI tool usage, third-party dependencies, and future commercial release readiness.

This is project documentation, not legal advice. It is meant to help track where code, audio, visual assets, and external packages came from so the project is easier to review before web, iOS, Android, or other commercial release.

Keep proof outside the repo too:

- AI tool subscription receipts and billing records.
- Original asset downloads and export files.
- Prompt text, generation links, IDs, and screenshots when available.
- Canva or other design tool receipts, account plan details, and terms snapshots.
- Third-party license notices if any package requires attribution.

Visual asset records should distinguish between the original generation tool and later editing or export tools. For this project, the current background assets were generated with ChatGPT / OpenAI image generation and then edited/exported in Canva for transparency/background removal.

## When to Update These Logs

Update the relevant legal/IP log when any of these happen:

- A new sound is added.
- A new visual asset is added.
- A new AI tool is used.
- Canva editing, background removal, or export is used.
- A new dependency is added.
- A new marketing or video asset is created.
- App store, analytics, ads, monetization, or other commercial tooling is added later.

Before a major commercial launch, consider asking an IP or game lawyer to review the asset logs, dependency licenses, app name, store copy, privacy policy, and monetization plans.
